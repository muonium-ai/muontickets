## Goal
Remove Python 3.14+ deprecation warnings caused by `datetime.utcnow()` while preserving MuonTickets timestamp wire formats and queue semantics.

## Issue Reported
- MuonTickets emitted `datetime.datetime.utcnow() is deprecated and scheduled for removal in a future version` on Python 3.14.3.
- The warning was observed during normal CLI workflows that update ticket timestamps and lease metadata.
- Affected paths included `now_compact()`, `now_utc_iso()`, `lease_expired()`, queue age scoring, and claimed lease counting.

## Analysis
- The implementation mixed serialized UTC strings with internally naive UTC datetimes.
- `parse_utc_iso()` converted aware timestamps back to naive UTC via `.replace(tzinfo=None)`, which kept comparisons working but preserved the deprecated model.
- Several runtime comparisons and age calculations called `datetime.utcnow()` directly instead of using a single normalized UTC helper.

## Fix
- Added `utc_now()` and `ensure_utc_aware()` in `muontickets/mt.py` to centralize UTC-aware datetime handling.
- Replaced all runtime `datetime.utcnow()` usages with aware UTC datetimes.
- Updated `parse_utc_iso()` to return aware UTC datetimes consistently.
- Kept external timestamp serialization backward compatible by continuing to emit ISO-8601 strings with trailing `Z`.
- Updated lease-expiry and queue scoring paths to compare aware UTC datetimes consistently.

## Verification
- Added regression tests in `tests/test_datetime_utc.py` covering timestamp generation, UTC parsing, lease expiry normalization, age scoring, and CLI warning-free execution.
- Re-ran queue allocation tests to ensure lease and age calculations still behave correctly.
- Verified there are no remaining runtime `utcnow()` usages in the Python codebase.
