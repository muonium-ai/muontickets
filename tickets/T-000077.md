## Goal
Make the Zig port build from current source on Zig 0.15 and restore the skipped Zig version tests so parity is fully green again.

## Issue Reported
- Full-suite testing showed two skipped Zig version tests because `zig build` failed in `ports/zig-mt`.
- The immediate build failure came from `build.zig.zon`, which Zig 0.15 rejected with `expected enum literal`.
- Even before rebuilding from source, the existing `zig-out/bin/mt-zig` binary was not reliable for `version --json` verification in the current workspace.

## Analysis
- The Zig port had drifted behind the installed Zig 0.15.2 toolchain in three places:
- `build.zig.zon` still used older manifest syntax for `.name`.
- `build.zig` still used older executable configuration fields such as `root_source_file`.
- `src/main.zig` still used older stdlib APIs for stdout writes, file writes, and `ArrayList` semantics.

## Fix
- Updated `ports/zig-mt/build.zig.zon` to Zig 0.15-compatible manifest syntax.
- Updated `ports/zig-mt/build.zig` to use `root_module = b.createModule(...)` and compatible path handling.
- Updated `ports/zig-mt/src/main.zig` to use current stdout and file-write APIs.
- Migrated list usage in the Zig port to managed `ArrayList` variants so the original append/deinit behavior remains valid on Zig 0.15.

## Verification
- `zig build -Doptimize=ReleaseSafe` now succeeds in `ports/zig-mt`.
- `ports/zig-mt/zig-out/bin/mt-zig version --json` now succeeds and reports version `1.0.1`.
